What is Bitcoin Potato?

Bitcoin Potato is an open source game of virtual Hot Potato using Bitcoin.

How does it work?

The best way to explain is via an example:

Let's say the current minimal fee is 1 BTC. To play, you send 1 BTC to the address displayed on the homepage. The next fee is adjusted to be 20% more = 1.2 BTC.

Sometime in the future, the next player will find Bitcoin Potato and send 1.2 BTC. 99% of this 1.2 BTC will go to you, as the last player, with the house taking 1% = 0.012 BTC as commission. At this point, you have earned a 19% interest on your "investment".

Sometime after that, another player will find this site and send at least 1.44 BTC, so the previous player will be compensated by at least 19%. This can go on for some time, until "the last player". This player (a.k.a the sucker), will not find anyone to follow him, and will actually lose all his money.

I sent the money, but the homepage haven't updated yet

We will wait until at least your transaction has at least one confirmation.

Can I send more than 20% more than the previous player?

You sure can. The same rules apply - 99% of your payment goes to the previous player, 1% goes to the house. The next player will have to pay 20% more than your payment to join.

Is this a pyramid scheme?

Well, sort of. It's not really a pyramid because the branching factor is 1, but it does depend on bringing more people in - this is not really a legitimate investment, but a game.

Only "invest" money you are willing to lose.

Still, what about security exploits?

I am not a security expert. Still, I believe the site is "relatively secure", for these reasons:

  • The site is completely open source, so you can check it for yourself (please report any breaches to ron.gross@gmail.com).
  • I do not store any Bitcoins on site. Your payment is immediately sent to the previous player, so nobody can steal my wallet.
  • It is stored on Heroku, which I believe to be "reasonably secure" (citation needed). I don't think an attacker will be easily able to get to my database. I am using strong passwords, my machine is not trojan infected, etc...
  • The code is written in a modern programming language (Java), using a good web framework (Play!), and to the best of my knowledge doesn't do anything "risky" that would be open to attack (no user input is used in SQL queries, etc...).
  • Security through obscurity - While a dedicate attacker might find a way to attack Bitcoin Potato, he doesn't really have that much to gain. At this time, the top fee on Bitcoin Potato is ${transaction.nextMinimalAmount} BTC - hardly an amount worth attacking a website for.

Why have you built this site?

Well, it's not in order to get rich. As I wrote above, I don't really expect the fees to pass 100 BTC (we're starting the game at 0.01 BTC, so we need about 50 rounds to get to 100 BTC, if everybody pays the minimal fee). I wrote it to:

  • Explore an idea and see if people will play
  • Learn how to write a webapp that transacts in Bitcoin
  • Provide an open source sample of such a site - the site is completely open source. Hopefully this will help other people build more serious Bitcoin enabled website.
